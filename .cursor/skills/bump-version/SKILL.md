# Bump Version

Update the application version consistently across every file that references it,
then push a git tag to trigger CI/CD.

## Files to Update

Update the **bare semver** (e.g. `0.1.0`) in these files:

| # | File | Field / Pattern |
|---|------|-----------------|
| 1 | `package.json` | `"version": "X.Y.Z"` |
| 2 | `src-tauri/Cargo.toml` | `version = "X.Y.Z"` (under `[package]`) |
| 3 | `src-tauri/tauri.conf.json` | `"version": "X.Y.Z"` |

Update the **display string** `DataZen vX.Y.Z` in these files:

| # | File | Pattern |
|---|------|---------|
| 4 | `src/windows/main/MainWindow.tsx` | `DataZen vX.Y.Z` in StatusBar |
| 5 | `src/windows/data-sync/DataSyncWindow.tsx` | `DataZen vX.Y.Z` in StatusBar |
| 6 | `e2e/specs/homepage-features.ts` | `DataZen vX.Y.Z` in test assertion |
| 7 | `docs/screenshots/mockup.html` | `DataZen vX.Y.Z` |
| 8 | `docs/prototypes/main-window-empty.svg` | `DataZen vX.Y.Z` |
| 9 | `docs/prototypes/main-window.svg` | `DataZen vX.Y.Z` |

Auto-generated (do NOT edit manually):

| # | File | Notes |
|---|------|-------|
| 10 | `src-tauri/Cargo.lock` | Regenerated by `cargo check` after Cargo.toml change |

## Step-by-Step Procedure

1. **Read current version** from `package.json` → `OLD`.
2. **Determine new version** `NEW` from user input (strip leading `v` if present).
3. **Edit files 1–9** above, replacing `OLD` → `NEW` (bare semver) and
   `DataZen vOLD` → `DataZen vNEW` (display strings).
4. **Regenerate Cargo.lock**: run `cargo check` in `src-tauri/`.
5. **Verify**: run `rg "OLD"` in the repo root to confirm no stale references remain
   (ignore transitive dependencies in lock files and `node_modules`).
6. **Commit** all changes with message: `chore: bump version to vNEW`.
7. **Tag & push** to trigger CI:

```bash
git tag "vNEW"
git push origin main --tags
```

## Handling Existing GitHub Release

If the tag `vNEW` already exists on the remote:

```bash
# Delete remote tag
git push origin :refs/tags/vNEW
# Delete local tag
git tag -d vNEW
# Re-create and push
git tag vNEW
git push origin main --tags
```

This forces GitHub Actions to re-run the release workflow and update the
existing release artifacts.
